What is MARMEAS?

MARMEAS is an abbreviation for MARking and MEAsurement on objects on images

MARMEAS is a handy tool is to identify objects on an image and let the application do geometric or photometric measurements on those objects.

MARMEAS allows to accelerate computer vision projects that lead to process automation:

Assume the following situation where images are already acquired from a production process but the objects of interest, the objects to detect, measure and check against are not known yet. The MARMEAS tool allows to manually mark those object on the image and use those markings as a reference database (ground truth) for computer vision. The reference database will be probably split apart:

  • Primarily to a training database , for the sake of teaching computer vision methods/parameters to achive optimal measurement and quality check performance.(Tuning aspect)
  • Secondarily to a test database used to proof the image analytical performance on an independet test set. (Generalization aspect)
Alternatively, any other means where objects need to be interactively marked and object features need to be determined and visualized this MARMEAS tool might be of interest.

Application Scenarios

Lets call our users by name just to indicate that they are different persons with different intentions.
  • Alice investigates a new image ad-hoc

    Alice just want a quick idea what characteristics a certain object on an image has. Therefore Alice selects an appropriate object model e.g. a line and marks the start and end point. The system computes the intensity profile, the line length and other measure for Alice.

  • Bob visually examines the objects and measurements, that are made available for an image.

    Bob loads the image and also the marked objects. The system visualizes the geometric positions of the objects and Bob can select any object he is interested in and the system will present the relevant measures. Bob can add new or remove existing measurements on that object.

  • Alice marks consistently objects for determining ground truth.

    Alice loads an image and marks on each image a set of objects that are important to know later on. This could be for development of an object detection algorithm, following the learing from data approach. Once she is done with one image Alice saves all the markings and measurements in a machine readable (xml) file. The repeats her marking activities on a lot of images of the same kind. Alice uses image templates and object templates to automate the manual marking as much as possible.

  • Bob marks consistently objects of the same kind on same kind of images for validating purposes.

    This is the same as the activity as before, exect that Bob saves his marked objects on a different place than Alice. Thus, validation cross checking can be done automatically (outside of this application) by any suitable validation script.

Major Features

  • Unit of Measurements as of the original Objects

    The image context allows to maintain context information for images, among that are the original unit of measurements of the objects as well as a conversion factor.

  • Model driven Marking

    Objects are described by an object model , like a point, a line, a circle, an rectangle and so on, that fit most the shape of the object. The users selects the model and marks characteristics points of that model. Those characteristic points allow to compute the model parameters, to completely describe the location, size and shape of the object as compact as possible.

  • Variety of supported Object Models

    There are object models for the most common shapes. Among them point, line circle. The user can derive custom models by creating object templates .

  • Customizable Set of Measurments on Objects

    The user can choose on an object bases which measurements he/she is actually interested in.

  • Variety of geometric Measurments

    Dependent on the applied object model , specific measurements based on the object model/object shape and the concrete markings of the object of interest are available. Those measurements are figures like length, width, height, geometric midpoint, circumfence, surface area, roundness

  • Flexible Areas of Interest per Object

    Areas of interest define the region on an image where intensity based measurements are taken for an object of interest. Therefore, areas of interest are bound to shape of the object of interest. The AoI's are also customizable on object bases. Depending on the object model multiple areas of Interest can be available.

  • Variety of monochrome Intensity Measurements

    Intensity Measurements are done on object of interest. Those measurement include intensity profiles as well as histograms. On either data, statistical figures are available like mean, variance/standard deviation, minimum, maximum, center of gravity, median, lower quartile, upper quartile

  • Machine readable XML output

    Object information including the related measurement and image context information are stored on machine readable files in a XML format (called xml object description - xod)

    Those XML representation can be read later again to visualize and browse previously done object markings and measurements.

  • Intuitive User Interaction

    The application provides a state of the art user interface, that is easy to use and easy to get into. Figure out by yourself. Any feedback or suggestion for improvement is welcome.

  • Visual marking/object/measurement presentation

    Compact presentation of an object and all related information as the object shape, the object on the image, the key figures and measurement as well as histogram/profile plots.

    Easy switching to another object or adding/modifying/removing of objects.

Project Roadmap

The roadmap lists anticipated releases and features. That is wishful thinking about the future. Although it might change quite often, it is the only communicated indicator of what to expect in the future. Of course, you can influence by getting involved into the development.

Version 1.x

As described above

Version 2.x

  • Extensibility of supported object models The user can derive custom models by creating object templates .
  • More supported base objects like triangle, polygon, ellipse

Version 3.x

  • False Color Maps to visualize small banded intensity distributions on objects of interest
  • Gradient Computation fits a linear functions and computes their parameter as well as quality of fit. This is useful to get an idea about a global gradient on a line object
  • Variety of color based intensity measurements to be worked out later

Version 4.x

  • Template based Automatic Object Location Estimation allows semi- automated object markings for known object. This is to get the finger into the pie for automated quality checks based on image at the context industrial automation
  • Objects of Objects allow measure distances, moments etc among objects. Multiple object can selected and grouped to a single object.

Technical Notes for Developers

MARMEAS is written in C++ facilitating the Qt4 library and tools (Designer, ...)

MARMEAS is is intended to be portable on Windows and Linux platforms. Other platforms might follow if somebody is willing to do the port. The platform specifics is considered as late as possible. The code is intended to be platform neutral as much as possible.

Cmake is used to control the build, install and package process. On windows NullSoft Installer is used, on linux debian packages are created. All that controlled by cmake.

Unit tests are maintained as well facilitating the cppunit test framework. Of course, those unit tests are there to assure functional correctness.

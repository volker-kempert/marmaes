Glossary

Base Object Type
Supported 2d object patterns including marking definitions, object model and superset of possible measurements

Image Context
definitions describing a image with respect to the situation it was taken and the circumstances it is of a value like real units of measurement, retrieval into a repository (through customised parameters, and directories)

There is a quick access i.e. empty context to speed-up ad-hoc assessments of images.

Image Context Template
Patterns that simplify the create of Image Contexts.
Marking Point
Is a point set by the user on a designated location of the surface of an object, that allows to derive some information about the object shape. I.e. this point is used to compute Object Model parameters.
Measurement
Any kind of geometric measurement (like size, max dimension, ...) or intensity (like average intensity, variance of intensity) measurement done on the object. Photometric measurement is a synonym to photometric measurement
Model Parameter
are the paramter that describe an object model in a way that shape, location, ..., i.e. all geometric features of the object can be determined.
Object
Is a representation of a recognisable 2d imaginary structure, that can be distinguished from other 2d imaginary structures. it consists of the object model, geometric parameters, markings information, name and notes and a set of shape and intensity measurements
Object Data File
Is a xml-formatted data/file containing marked objects including their object model and their associated measurements The related image context . is included as well.

It is recommended to use the extension 'xod'.

Object Fitter
Is the facilitating method that allows to compute the model parameter for a given model from the set of given marking points. It is possible to have multiple Object Fitter for a single Object Model. Those fitter consider different sets of marking points.
Object Model
Is a geometric figure that looks similar to the shape of the object. The parameters of the figure (e.g. the radius in case the model is a circle) are fitted, such that object model and object shape appear as similar as possible.
Object Marking
Is the activity to determine marking points on the shape of the object, that are used to compute model parameter that fit the object model to the object.
Object Shape
Data describing the 2d structure from a topological point of view, it does not consider intensity information. Model based shapes are characterised throughout a set of model dependent parameter
Object Texture
Data describing the 2d structure considering all intensity information (either grey-scale or colour channels).
Object Type
is a Base Object Type extended by a name (representing the business context) and reduction to subset of relevant measurements ( The subset of measurements is derived from measurements of the referenced base object type)
Object Template
Are used to create Objects of a specific Objects Type. I.e. Object Templates define an Object Type.
